Etiquetas Html5

Mejores paginas para diseño web.

Etiquetas de metadatos, seccion y modelo de cajas.

1Etiquetas de metadatos

NOTA: Metadato significa datos sobre los datos.

A) <head>: Representa una colección de metadatos para el documento. Es un contenedor para las etiquetas: title, style, meta, link, script y base.

B) <title>: La etiqueta title es un fragmento de código HTML en el que se incluye el título de una página web, que normalmente es un texto que describe lo que encontraremos dentro de esa página.

C) <base>: Especifica la URL base y/o destino para todas las URL relativas en una página. Esta etiqueta debe tener: un href + un atributo de destino. base href=”https:www.prueba.com/ “ target=”_blanck”

D) <link rel>: Define la relación entre el documento actual y el recurso externo. Ej: importar hojas de estilo CSS, favicon... rel: información del tipo de archivo que agregamos.

E) <style>: Es el elemento encargado de indicar la información de estilo. Sus etiquetas son: style y /style

F) <script>: Escribe código JS dentro del documento HTML. Poco recomendable.

G)<meta>

Las etiquetas <meta>: sireven para:
  • charset=”UTF8”-----> Especificar el conjunto de caracteres
  • meta name="descripcion" content="Paginas web de tutoriales" --->Descripción de la página
  • meta name=”keywords” content=”HTML, CSS, Javascript”---->Palabras clave
  • meta name=”author” content=”Diego”----->autor de la página
  • meta http-equiv=”refresh” content=”30”

2Etiquetas de sección

A)<body>: Todo el contenido de la página web. Lo que va a ver el usuario.

B)<header>: Representa la cabecera de un bloque de contenido. Dentro del body lo primero es la cabecera del documento. Ej: título, menú navegación superior. Esta etiqueta no hace nada de apariencia en el navegador del PC, si en otros dispositivos. Útil para que nuestra página sea compatible con todos los dispositivos del mercado.

C)<article>: Representa un contenido completo dentro de una sección. Se recomienda que dentro exista un encabezado.

D)<h1>...<h6>: Es un título de una sección o bloque. También indica la importancia del bloque h1, h2... Diferente tamaño y siempre en negrita. Los buscadores buscan mucho estas etiquetas hx. Se supone que la información que contienen es importante.

E)<section>: Divide un bloque de contenido. Ej: si un artículo tiene dos párrafos y cada párrafo habla de una cosa.

F)<aside>: Representa un contenido relacionado con el contenido principal. Usado para menús laterales. Indica al navegador que va a haber un contenido de barra lateral

G)<footer>: Representa el pie de un bloque de contenido. Pie de página: horario, dirección, datos de contacto… El HEADER y el FOOTER no solo se usa para cabecera y pie del documento principal. También para definir cabecera y pie de otros contenidos de la web.

H)<nav>: Representa los enlaces de navegación de la página.

2Modelo de cajas

En el modelo de cajas estándar, cuando estableces los atributos width y height para una caja, defines el ancho y el alto del contenido de la caja. Cualquier área de relleno y borde se añade a ese ancho y alto para obtener el tamaño total que ocupa la caja. Esto se muestra en la imagen que encontrarás a continuación.
